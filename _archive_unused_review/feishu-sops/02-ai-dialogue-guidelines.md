# 02｜项目中与 AI 对话的注意事项与正确方式

适用对象：使用 AI 辅助维护、优化、扩展个人作品集网站的设计师。

核心原则：不要把 AI 当成“许愿工具”，要把它当成“会写代码的协作设计师”。你给的信息越具体，它越能做出稳定、可维护、符合审美方向的结果。

## 一、和 AI 对话前先做三件事

### 1. 先说明项目性质

不要只说“帮我美化网页”。要说明这个网站服务谁、看的人关心什么。

推荐说法：

```txt
这是一个面向招聘方的个人视觉设计作品集网站，目标是展示成熟商业视觉能力、项目执行力和 AIGC 工作流。整体需要克制、精致、图像主导，不能做成科技感模板或简历页面。
```

### 2. 先要求 AI 阅读项目规范

当前项目有三个关键文件：

- `PRODUCT.md`
- `DESIGN.md`
- `AGENTS.md`

推荐说法：

```txt
请先阅读 PRODUCT.md、DESIGN.md 和 AGENTS.md，再开始修改。所有设计和代码都要遵守这三个文件里的定位、视觉规则和工作方式。
```

### 3. 先明确改动范围

范围越清楚，AI 越不容易乱改。

推荐说法：

```txt
这次只修改首页第一屏和精选项目区域，不要改项目数据，不要改路由，不要新增依赖。改完后运行 npm run build。
```

## 二、不要这样和 AI 对话

### 错误 1：目标太空

不推荐：

```txt
帮我做高级一点。
```

问题：高级是什么？更大字、更黑、更贵、更克制，AI 不知道。

推荐：

```txt
请把项目详情页优化得更像高级商业案例展示。重点提升 hero 图的视觉冲击、元信息条的秩序感、画廊间距和标题层级。不要使用发光、渐变和过圆卡片。
```

### 错误 2：一次让 AI 改太多

不推荐：

```txt
帮我把整个网站全部改好。
```

问题：范围太大，容易破坏已有结构。

推荐：

```txt
第一步先检查首页的信息层级和视觉问题，只给我 5 条最重要的修改建议。确认后再开始写代码。
```

或者：

```txt
这次只优化 ProjectDetail.jsx 和相关 CSS，不要改其他页面。
```

### 错误 3：只给审美词，不给边界

不推荐：

```txt
做得更有设计感。
```

推荐：

```txt
请增强设计感，但方向是成熟商业视觉，不是科技感。可以提升留白、图片比例、字体层级、hover 反馈和移动端节奏。不要加入霓虹、玻璃拟态、复杂背景、夸张动效。
```

### 错误 4：让 AI 直接替你决定内容真实性

不推荐：

```txt
帮我把项目经历写得厉害一点。
```

推荐：

```txt
请在不夸大事实的前提下，帮我把这段项目经历改得更适合作品集展示。强调我真实负责过的视觉设计、画面合成、AIGC 生成、筛选、修图和交付工作。
```

### 错误 5：没有要求验证

不推荐：

```txt
改完告诉我。
```

推荐：

```txt
改完后请运行 npm run build，并说明修改了哪些文件、验证结果是什么、还有哪些风险需要我人工检查。
```

## 三、正确对话公式

每次找 AI 改网站，可以按这个格式写：

```txt
背景：
这是面向招聘方的视觉设计作品集，定位是成熟、商业、精致、图像主导。

请先阅读：
PRODUCT.md、DESIGN.md、AGENTS.md。

这次目标：
优化首页精选项目轮播，让它更突出项目质感。

改动范围：
可以改 src/components/FeaturedCarousel.jsx 和 src/index.css。
不要改 src/data/projects.js，不要改路由，不要新增依赖。

视觉要求：
Apple-like 但不复制 Apple；黑白灰为主，橙色少量强调；小到中等圆角；动效要慢、轻、克制。

验收标准：
桌面端和手机端都不能文字重叠；图片不能被不合理裁切；hover 和轮播状态自然；最后运行 npm run build。
```

## 四、不同任务的推荐提示词

### 1. 新增项目

```txt
我要新增一个作品项目。请只修改 src/data/projects.js，保持当前数据结构和字段命名。项目资料如下：[粘贴项目名称、分类、年份、角色、工具、背景、职责、图片路径]。请帮我补齐 project 对象，并检查 titleLines、coverImage、heroImage、images 是否合理。
```

### 2. 优化首页

```txt
请优化首页，让招聘方第一眼更明确看到我是视觉设计师和 AIGC 内容创作者。请保留当前 SphereCarousel 的作品图结构，优化文字层级、间距、滚动动效和移动端表现。不要做营销落地页，不要新增大段说明文字。
```

### 3. 优化项目详情页

```txt
请优化项目详情页的案例展示感。重点处理 hero 首图、项目元信息、画廊布局、项目背景和职责列表。不同类型项目可以保留不同画廊比例。不要把详情页做成表格或简历式信息堆叠。
```

### 4. 修复移动端

```txt
请专门检查移动端 375px 宽度下的布局问题。重点看标题换行、按钮宽度、项目卡片、详情页图文间距、导航可点击性。修复时尽量只改 CSS 和 Tailwind class，不要重构组件。
```

### 5. 图片加载优化

```txt
请检查当前图片加载和 srcSet 策略。目标是首屏快、画廊清晰、移动端不要加载过大图片。优先使用已有 LazyImage、projectImageSrcSet、projectGallerySrc 和图片处理脚本，不要引入新图片库。
```

### 6. 文案润色

```txt
请把以下项目文案润色成适合作品集的表达。要求真实、不夸张、商业感强、简洁。避免空泛词，比如“赋能”“打造极致体验”。输出 title、description、background、responsibilities 四部分。
```

### 7. 构建报错修复

```txt
npm run build 报错如下：[粘贴报错]。请先判断是语法、路径、依赖还是图片处理问题，再做最小范围修复。不要顺手重构无关文件。
```

## 五、给 AI 反馈时怎么说

AI 第一次修改不一定完全符合审美，需要用“具体反馈”继续迭代。

不推荐：

```txt
不够好看。
```

推荐：

```txt
现在的问题是：1. 首页标题过大，压住了作品图；2. 橙色面积太多，显得不够高级；3. 移动端项目卡片间距太小。请只针对这三个问题修改。
```

不推荐：

```txt
再高级一点。
```

推荐：

```txt
请减少装饰感，增加留白，让图片更像商业广告案例展示。按钮可以更小、更克制，文字层级更清楚。
```

## 六、每次对话都要守住的边界

### 1. 内容真实性

项目经历不能让 AI 编造。可以润色，但不能虚构品牌、职责、数据、职位。

### 2. 结构稳定

当前项目内容维护很简单，核心是 `src/data/projects.js`。不要轻易让 AI 改成复杂后台、CMS 或数据库。

### 3. 审美一致

任何新增页面和组件都要符合：

- 成熟商业视觉
- 图像主导
- 克制动效
- 小到中等圆角
- 黑白灰为主
- 橙色少量点缀

### 4. 每次改完要验证

至少要求 AI 做：

```bash
npm run build
```

需要人工再看：

- 首页第一屏。
- 作品页。
- 至少 2 个项目详情页。
- 手机端。
- 简历下载。
- 联系方式。

## 七、推荐的长期协作方式

### 小步快跑

一次只改一个区域：

- 首页首屏
- 精选项目
- 作品卡片
- 项目详情页
- 关于页
- 移动端
- 图片性能

### 先审美，后代码

如果你不确定方向，先让 AI 出方案，不要马上写代码：

```txt
先不要改代码。请基于 PRODUCT.md 和 DESIGN.md，给我 3 个首页优化方向，每个方向说明优点、风险和适合修改的文件。
```

### 让 AI 解释修改

改完后要求：

```txt
请用设计师能理解的方式说明这次修改：解决了什么视觉问题、改了哪些文件、以后我维护时要注意什么。
```

### 保存好成功提示词

以后每次 AI 做出满意结果，把提示词保存下来，形成自己的提示词库。
