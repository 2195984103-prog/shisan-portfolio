# Agent Notes

This is a visual design portfolio for recruiters. Before making frontend changes, read:

- `PRODUCT.md` for positioning, audience, and anti-references.
- `DESIGN.md` for visual rules and the Apple-like portfolio direction.

## Working Style

Preserve the existing React, Vite, and Tailwind structure. The content model is intentionally simple: project data lives in `src/data/projects.js`, and assets live in `public/assets`.

When refining UI, prioritize:

- mature commercial visual design
- precise spacing and alignment
- small-to-medium radii
- image-led project presentation
- calm Apple-like polish without copying Apple
- strong responsive behavior

Avoid:

- flashy technology aesthetics
- overly rounded cards
- resume-like dense layouts
- social-media template styling
- decorative gradients, glow, or noisy effects
- broad rewrites that make content harder to update

Run `npm run build` after code changes.
