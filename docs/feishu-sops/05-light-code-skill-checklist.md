# 05｜需要补齐的能力清单（轻代码）

目标：不要求成为前端工程师，但要能看懂作品集项目结构，能和 AI 高质量协作，能自己完成常见维护。

建议学习方式：每个能力只学到“能维护自己的作品集”为止，不需要深入到职业程序员水平。

## 一、必须掌握：文件和项目结构

### 需要会什么

- 能找到项目目录。
- 能区分 `src`、`public`、`docs`、`scripts`。
- 能知道哪些文件能改，哪些文件少改。
- 能看懂图片路径。

### 当前项目重点

| 位置 | 你需要知道 |
| --- | --- |
| `src/data/projects.js` | 所有作品内容 |
| `public/assets/projects` | 原始项目图片和视频 |
| `src/pages` | 页面 |
| `src/components` | 页面里的组件 |
| `src/index.css` | 全局样式和动效 |
| `package.json` | 项目命令 |

### 练习任务

- 找到任意一个项目的 `coverImage`。
- 找到这个项目对应的图片文件夹。
- 修改一个项目的 `description`。
- 本地预览确认页面变化。

## 二、必须掌握：终端基础命令

### 需要会什么

| 命令 | 用途 |
| --- | --- |
| `cd` | 进入项目目录 |
| `npm run dev` | 启动本地预览 |
| `npm run build` | 检查能否上线 |
| `npm run process-images` | 处理图片 |
| `Control + C` | 停止本地服务 |

### 当前项目常用流程

```bash
cd "/Users/shisan/Documents/作品集 4"
npm run dev
```

改完后：

```bash
npm run build
```

### 练习任务

- 启动一次本地网站。
- 停止本地网站。
- 运行一次构建。
- 看懂构建成功和失败的大概区别。

## 三、必须掌握：JS 对象基础

### 为什么要学

项目数据写在 `src/data/projects.js`，本质是一组 JavaScript 对象。你不需要会写复杂 JS，但要能改字段。

### 需要会什么

- 字符串用引号包起来。
- 数组用 `[]`。
- 对象用 `{}`。
- 每个字段之间用逗号。
- 不能漏掉引号、逗号、括号。

### 当前项目示例

```js
{
  id: "la-mer-aigc-visual",
  title: "海蓝之谜 AIGC 创意视觉",
  tools: ["ComfyUI", "Stable Diffusion", "Photoshop"],
  responsibilities: [
    "拆解品牌调性与美妆视觉关键词",
    "编写并优化 AIGC 生成提示词"
  ]
}
```

### 练习任务

- 新增一个 `responsibilities` 条目。
- 修改 `tools` 里的工具名称。
- 修改 `year`。
- 故意漏一个逗号，观察 `npm run build` 报错，再修回来。

## 四、必须掌握：图片路径和命名

### 需要会什么

- 知道 `public` 目录里的文件在代码里不写 `public`。
- 知道路径要从 `/assets/...` 开始。
- 知道大小写要完全一致。
- 知道图片替换后要重新处理。

### 示例

真实文件：

```txt
public/assets/projects/dongfeng-lantu-kv/cover.jpg
```

代码路径：

```js
coverImage: "/assets/projects/dongfeng-lantu-kv/cover.jpg"
```

### 练习任务

- 替换一个项目的 `cover.jpg`。
- 运行 `npm run process-images`。
- 检查首页卡片是否更新。

## 五、应该掌握：HTML 和 JSX 基础

### 为什么要学

页面文件是 React JSX，看起来像 HTML。你需要能识别标题、段落、链接、按钮、图片。

### 需要认识

| 写法 | 含义 |
| --- | --- |
| `<section>` | 页面区块 |
| `<div>` | 容器 |
| `<p>` | 段落 |
| `<h1>` / `<h2>` | 标题 |
| `<a href="">` | 链接 |
| `<img src="">` | 图片 |
| `className=""` | 样式类 |

### 练习任务

- 在 `Home.jsx` 找到联系区。
- 修改按钮文字。
- 修改邮箱或电话。
- 不改动其他组件结构。

## 六、应该掌握：Tailwind CSS 基础

### 为什么要学

当前项目大量样式写在 `className` 里，例如：

```jsx
<div className="grid gap-4 md:grid-cols-3 lg:gap-5">
```

### 需要认识

| 类名 | 大概意思 |
| --- | --- |
| `grid` | 网格布局 |
| `flex` | 弹性布局 |
| `gap-4` | 间距 |
| `px-5` | 左右内边距 |
| `py-5` | 上下内边距 |
| `text-white/70` | 白色 70% 透明度 |
| `bg-black/20` | 黑色 20% 透明度 |
| `md:grid-cols-3` | 中等屏幕开始三列 |
| `lg:gap-5` | 大屏幕使用更大间距 |

### 练习任务

- 把一个区块的 `gap-4` 改成 `gap-5` 看变化。
- 把 `md:grid-cols-3` 改成 `md:grid-cols-2` 看变化。
- 理解 `sm:`、`md:`、`lg:` 是响应式断点。

## 七、应该掌握：CSS 变量和全局样式

### 当前项目重要变量

在 `src/index.css`：

```css
--color-accent: #f26a1b;
--radius-control: 10px;
--radius-media: 16px;
--radius-panel: 18px;
--space-section-y: clamp(5rem, 7vw, 8.5rem);
```

### 需要会什么

- 知道改变量会影响全站。
- 知道 `clamp()` 是响应式尺寸。
- 知道圆角不要改得太大。
- 知道强调色不要大面积使用。

### 练习任务

- 找到 `--color-accent`。
- 找到 `--radius-media`。
- 不一定要修改，但要知道它们控制什么。

## 八、应该掌握：响应式检查

### 需要会什么

- 会缩小浏览器宽度检查手机端。
- 会用浏览器开发者工具切换尺寸。
- 会检查 375px、768px、1440px。

### 检查重点

- 标题是否溢出。
- 图片是否乱裁。
- 按钮是否太窄。
- 导航是否可点。
- 页面是否横向滚动。

### 练习任务

- 在手机宽度打开首页。
- 打开一个项目详情页。
- 找出 1 个你觉得间距不舒服的位置，并让 AI 修复。

## 九、应该掌握：构建报错阅读

### 你不需要完全看懂，但要能判断方向

常见类型：

| 报错关键词 | 可能原因 |
| --- | --- |
| `Unexpected token` | 少逗号、括号、引号 |
| `Cannot find module` | 文件路径或依赖错误 |
| `Failed to load` | 图片或资源路径错误 |
| `Build failed` | 构建失败，需要看上方具体报错 |

### 给 AI 的正确方式

```txt
npm run build 报错如下。我不确定原因，请你判断并做最小范围修复，不要重构无关文件。
[粘贴完整报错]
```

## 十、建议掌握：Git 基础

如果后续经常迭代网站，建议学习 Git 的最低限度用法。

### 需要知道

- Git 可以记录每次修改。
- 改坏了可以回看历史。
- 上线前最好确认改了哪些文件。

### 需要认识

| 命令 | 作用 |
| --- | --- |
| `git status` | 查看哪些文件改了 |
| `git diff` | 查看具体改了什么 |
| `git add` | 准备提交 |
| `git commit` | 保存一次修改记录 |

如果不熟，不要自己执行危险命令，例如：

```bash
git reset --hard
```

## 十一、建议掌握：AI 协作能力

这是轻代码维护最重要的能力。

### 你需要会

- 描述目标。
- 限定范围。
- 提供反面要求。
- 要求 AI 先读项目文件。
- 要求 AI 改完验证。
- 能根据结果继续给具体反馈。

### 推荐固定话术

```txt
请先阅读 PRODUCT.md、DESIGN.md 和 AGENTS.md。
这次只修改 [范围]。
目标是 [目标]。
不要 [反面要求]。
改完后运行 npm run build，并说明修改文件和验证结果。
```

## 十三、30 天轻代码补齐计划

### 第 1 周：会运行项目

- 学会进入项目目录。
- 学会启动本地预览。
- 学会停止服务。
- 学会运行构建。
- 熟悉文件结构。

### 第 2 周：会改内容

- 修改一个项目简介。
- 替换一张封面图。
- 新增一个职责条目。
- 调整首页精选项目顺序。

### 第 3 周：会改轻样式

- 认识 Tailwind 间距和栅格。
- 调整一个页面区块间距。
- 调整按钮文字。
- 检查手机端。
- 让 AI 修一个响应式问题。

### 第 4 周：会做完整维护

- 新增一个完整项目。
- 处理图片。
- 本地预览。
- 构建检查。
- 写清楚修改记录。

## 十四、最终目标

你不需要成为专业前端，但要做到：

- 能自己新增项目。
- 能自己替换图片。
- 能自己改基础文字。
- 能判断 AI 有没有乱改。
- 能把需求说清楚。
- 能完成上线前检查。

达到这个水平，就足够稳定维护个人作品集网站。
